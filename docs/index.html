Tutorial on using sitePath

Chengyang Ji

2019-02-13

Introduction

The sitePath package does hierarchical search for fixation events given multiple sequence alignment and phylogenetic tree. These fixation events can be specific to a phylogenetic lineages or shared by multiple lineages. This is achieved by three major steps:

  1. Import tree and sequence alignment
  2. Resolve phylogenetic lineages
  3. Hierarchical search for lineage-dependent fixation events

1. Import tree and sequence alignment

Thereâ€™re various R packages for parsing phylogenetic tree and multiple sequence alignment files. For now, sitepath accepts phylo object and alignment object. Functions from ggtree and seqinr are able to handle most file formats.

1.1 Parsing tree

The S3 phylo class is a common data structure for phylogenetic analysis in R. The CRAN package ape provides basic parsing function for reading tree files. The Bioconductor package ggtree provides more comprehensive parsing utilities.

library(ape)

tree <- read.tree(system.file("extdata/ZIKV.newick", package = "sitePath"))

Tree must be rooted before further analysis

tree <- root(tree, "ANK57896")

1.2 Parsing sequence alignment

Most multiple sequence alignment format can be parsed by seqinr.

library(seqinr)

align <- read.alignment(system.file("extdata/ZIKV.fasta", package = "sitePath"), "fasta")

2. Resolve phylogenetic lineages

The names in tree and aligment must be matched. The fundamental approach in identifying phylogenetic lineages is trimming tree leaves/tips to expose the major branches. Before finding putative phylogenetic lineages, there involves a few more steps to evalute the impact of threshold on result.

2.1 The impact of threshold on resolving lineages

In the current version, the resolving function only takes sequence similarity as one single threshold. The impact of threshold depends on the tree topology hence there is no universal choice. The function sneakPeak samples thresholds and calculates the resulting number of paths. The use of this function can be of great help in choosing the threshold.

library(sitePath)

preassessment <- sneakPeek(tree, align)

2.2 You have to choose a threshold by yourself!

There might be one point where the number of resulting paths has an unusual drop and that could be the preferable similarity threshold. For example, here 0.996 is a reasonable choice. You can use the function sitePath to get a S3 sitePath class object1 for downstream analysis.

paths <- sitePath(tree, align, 0.996)
paths
#> 5 paths

3. Hierarchical search for fixation events

Now youâ€™re ready to find fixation events.

3.1 Fixation mutations

The hierarchical search is done by fixationSites function. The function outputs a list of mutations with the sequence names involved before and after the fixation.

fixations <- fixationSites(paths)
fixations
#> N1902H V763M T777M S139N Y2086H M1143V D3398E G894A M2074L R3045C 
#> No reference sequence specified. Using alignment numbering

3.2 Visualize the result

Use plot to visualize each fixation mutation. Youâ€™ll have to specify which mutation to be visualized. Because the names(fixationSites) are the mutation names, you can also use one of them as function input

par(mar = c(1,1,1,1))
plot(fixations, "S139N")
plot(fixations, names(fixations)[6])

Additional functions

The above concluded the tutorial. This part is extra and experimental. You may stop here if you wish.

Just using the trimming function

The function groupTips applies the same operation as the functionsitePath does but it outputs clusters of tree tips instead of paths

grouping <- groupTips(tree, align, 0.996)

Inspect one site

If you just input the number of the site, the function will plot distribution of amino acid on the tree

par(mar = c(1,1,1,1))
plot(fixations, 139)
plot(fixations, 763)

SNP sites

An SNP site could potentially undergo fixation event. The SNPsites function predicts possible SNP site and the result could be what youâ€™ll expect to be fixation mutation.

snps <- SNPsites(tree, align)
par(mar = c(1,1,1,1))
plot(fixations, snps[4])
plot(fixations, snps[5])
